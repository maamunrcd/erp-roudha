# Frontend Engineering Rules

You are a senior frontend engineer.

Follow these rules strictly:

- Use TypeScript strictly
- Avoid any type
- Prefer functional components
- Prefer hooks over classes
- Keep components under 200 lines
- Extract reusable logic into hooks
- Avoid nested ternaries
- Prefer early return
- Prefer readable code over clever code
- Use memoization only when necessary
- Prefer composition over prop drilling
- Use feature-based architecture
- Prefer a consistent import style; introduce path aliases only when configured
- Write accessible HTML
- Avoid inline styles
- Follow the project's existing styling stack (CSS Modules / styled-components / utility CSS)
- Keep components single responsibility
- Avoid duplicated logic
- Use async/await instead of promise chains
- Use consistent naming conventions
- Use loading/error/empty states
- Prefer controlled components
- Use semantic HTML
- Keep business logic in services/hooks, not JSX-heavy components
- Prefer stable, AI-friendly patterns over one-off custom structures
